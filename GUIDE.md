# دليل استخدام منصة مسارات التعليمية

## نظرة عامة

منصة مسارات التعليمية هي تطبيق ويب عربي متكامل يربط المعلمين والطلاب في بيئة تعليمية حديثة وآمنة. تم تصميم المنصة بالكامل باللغة العربية مع دعم RTL ووضع داكن افتراضي.

## المميزات الرئيسية

### 1. نظام الأدوار (RBAC)
- **المستخدمون العاديون**: يمكنهم تصفح الصفحة الرئيسية والملفات التعليمية بدون تسجيل دخول
- **المعلمون**: لديهم لوحة تحكم خاصة لرفع وإدارة الملفات التعليمية
- **الإداريون**: لديهم لوحة تحكم إدارية كاملة "لوحة التحكم" لإدارة المعلمين والمحتوى

### 2. لوحة التحكم الإدارية
- عرض إحصائيات شاملة (عدد المعلمين، الملفات، الطلبات)
- إدارة طلبات تسجيل المعلمين (موافقة/رفض)
- عرض جميع المعلمين المسجلين مع حالاتهم
- إدارة الملفات التعليمية
- نظام إشعارات متقدم للأحداث المهمة

### 3. لوحة تحكم المعلمين
- التسجيل كمعلم مع بيانات التخصص والخبرة
- رفع الملفات التعليمية إلى S3 مع بيانات وصفية
- عرض وإدارة الملفات المرفوعة
- حذف الملفات غير المرغوب فيها

### 4. نظام الإشعارات
- إشعارات تلقائية للإداريين عند تسجيل معلم جديد
- إشعارات للأحداث المهمة في النظام
- عداد للإشعارات غير المقروءة
- إمكانية تحديد الإشعارات كمقروءة

### 5. تخزين الملفات
- رفع الملفات التعليمية إلى S3 بشكل آمن
- روابط وصول مباشرة للملفات
- حفظ البيانات الوصفية في قاعدة البيانات

## إعداد Firebase

### الخطوة 1: إنشاء مشروع Firebase
1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. انقر على "إضافة مشروع" (Add Project)
3. أدخل اسم المشروع: `masarat-school`
4. اتبع الخطوات لإكمال إنشاء المشروع

### الخطوة 2: إعداد Firestore Database
1. من القائمة الجانبية، اختر "Firestore Database"
2. انقر على "إنشاء قاعدة بيانات" (Create Database)
3. اختر الموقع الجغرافي المناسب
4. ابدأ في وضع الإنتاج (Production Mode)
5. انسخ محتوى ملف `firestore.rules` وألصقه في قواعد Firestore

### الخطوة 3: إعداد Firebase Hosting
1. من القائمة الجانبية، اختر "Hosting"
2. انقر على "البدء" (Get Started)
3. قم بتثبيت Firebase CLI على جهازك:
   ```bash
   npm install -g firebase-tools
   ```
4. قم بتسجيل الدخول:
   ```bash
   firebase login
   ```
5. قم بتهيئة المشروع:
   ```bash
   firebase init
   ```
   - اختر Hosting و Firestore
   - اختر المشروع الذي أنشأته
   - استخدم `dist` كمجلد النشر
   - اختر "نعم" لإعداد SPA

### الخطوة 4: الحصول على بيانات الاعتماد
1. من إعدادات المشروع (Project Settings)
2. انتقل إلى "تطبيقاتك" (Your Apps)
3. انقر على أيقونة الويب (</>)
4. سجل التطبيق واحصل على بيانات الاعتماد:
   - API Key
   - Auth Domain
   - Project ID
   - Storage Bucket
   - Messaging Sender ID
   - App ID

### الخطوة 5: إضافة بيانات الاعتماد للمشروع
قم بإضافة المتغيرات التالية إلى ملف `.env`:
```
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

## النشر على Firebase Hosting

### 1. بناء المشروع
```bash
pnpm build
```

### 2. النشر
```bash
firebase deploy
```

سيتم نشر التطبيق على عنوان مثل:
`https://masarat-school.web.app`

## استخدام المنصة

### للمستخدمين العاديين
1. افتح الصفحة الرئيسية
2. تصفح الملفات التعليمية المتاحة
3. قم بتحميل الملفات التي تحتاجها

### للمعلمين
1. انقر على "تسجيل الدخول"
2. سجل الدخول باستخدام حساب Manus
3. انقر على "التسجيل كمعلم"
4. املأ بياناتك (التخصص، رقم الهاتف، نبذة)
5. انتظر موافقة الإدارة
6. بعد الموافقة، يمكنك رفع الملفات التعليمية

### للإداريين
1. سجل الدخول كإداري (يجب تعيين الدور من قاعدة البيانات)
2. انتقل إلى "لوحة التحكم"
3. راجع طلبات التسجيل الجديدة
4. وافق أو ارفض المعلمين
5. راقب الإحصائيات والملفات المرفوعة

## تعيين أول إداري

لتعيين أول إداري في النظام:

1. افتح لوحة إدارة قاعدة البيانات في Manus
2. ابحث عن المستخدم في جدول `users`
3. قم بتحديث حقل `role` إلى `admin`

أو استخدم SQL:
```sql
UPDATE users SET role = 'admin' WHERE email = 'admin@example.com';
```
